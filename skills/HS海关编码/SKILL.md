# HS海关编码系统

一个面向出口报关场景的智能HS编码归类与查询系统。采用"免手工录库"设计理念，优先自动同步与实时补查，确保查询结果的准确性和时效性。

## 🎯 核心功能

### 1. **智能商品归类**
- 根据文本描述或技术文档自动抽取商品属性
- 智能推荐中国出口申报HS编码
- 生成完整的报关要素

> ⚠️ **图片识别功能尚未实现**，目前仅支持文本/文档输入。

### 2. **实时数据查询**
- 查询出口退税率（退税率缓存 TTL 30 天，超期自动补查）
- 识别监管条件（缓存 TTL 60 天）
- 输出中国-印尼HS候选映射
- 缓存实时查询结果，标注来源和可信度

### 3. **多源数据整合**
- **缓存优先**：优先使用本地缓存数据（含 TTL 过期检查）
- **同步数据**：使用手动触发同步的税则规则（增量 upsert，不覆盖历史数据）
- **实时补查**：在线查询最新官方数据
- **人工复核**：保留人工确认入口

> ⚠️ **数据同步需手动触发**，不会自动每日执行。同步命令：
> ```bash
> python3 scripts/data_sync.py --source <文件> --target <hs|rebate|supervision|mapping>
> ```

## 📋 设计原则

### 1. **数据来源透明化**
- 不手工录库，优先自动同步与实时补查
- 所有查询结果必须标记来源、时间、可信度
- 实时查询结果必须做来源质量分级，优先官方来源

### 2. **查询流程标准化**
- 查询顺序固定：`cache -> synced -> live_lookup -> manual_review`
- 不得无来源猜测退税率、监管条件、印尼对应编码
- 图片识别尚未实现；所有归类结论均保留人工复核入口

### 3. **结果可信度分级**
- **official**：官方来源，最高可信度
- **reference**：参考来源，中等可信度
- **unknown**：未知来源，需要验证
- **low**：低可信度，需要人工复核

### 工作流设计：

#### 1. **输入路由** (`input_router`)
- 根据输入类型分流到文档解析、图片初判或文本查询
- 支持混合输入模式

#### 2. **技术资料解析** (`product_doc_parser`)
- 解析产品规格书、测试报告等技术文档
- 提取商品属性信息

#### 3. **图片初判** (`image_classifier`)
- 基于图片进行商品类别初步判断
- 提供视觉参考信息

#### 4. **商品信息标准化** (`normalize_product_info`)
- 标准化商品描述和属性
- 统一术语和格式

#### 5. **HS编码查询** (`hs_code_lookup`)
- 多级查询：缓存 -> 同步数据 -> 实时补查
- 返回候选HS编码列表

#### 6. **报关要素生成** (`declaration_elements`)
- 生成完整的报关要素
- 包括商品名称、用途、材质等

#### 7. **退税率查询** (`rebate_rate_lookup`)
- 查询出口退税率
- 标注来源和时间

#### 8. **监管条件识别** (`supervision_conditions`)
- 识别监管条件代码
- 提供所需单证清单

#### 9. **印尼编码映射** (`indonesia_mapping`)
- 查询中国-印尼HS编码对应关系
- 提供映射依据

#### 10. **结果整合与输出** (`result_integration`)
- 整合所有查询结果
- 生成最终输出报告
